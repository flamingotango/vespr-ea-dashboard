Section 11 What You're Building

You're building an AI system that reads every message across every platform you use, sorts them by urgency, drafts replies in your voice, and presents everything in a single dashboard each morning. You review, approve, and move on. Five minutes instead of three hours.

The System Architecture

Here's the flow: your messages come in across all channels. n8n catches every one of them and sends them to Claude. Claude reads the message, scores it by urgency (0-100), writes AI notes analyzing the context, and drafts a reply in your voice. Everything gets pushed to your dashboard where you review, edit if needed, and approve. One click sends the reply back through the original channel.

What this replaces

A full-time executive assistant costs $4,000-$8,000/month. This system costs under $300/month and works 24/7. It never misses a message, never forgets context, and gets smarter the more you use it.

Section 33 Setting Up Claude Pro Max

Claude is the brain of your entire system. It reads messages, understands context, scores urgency, and writes draft replies in your voice. The Pro Max plan gives you Claude Code, which is what you'll use to build everything.

Go to claude.ai and create an account

Use your primary email. This will be your AI command center.

Subscribe to the Pro Max plan ($100/month)

Click your profile icon in the bottom left, then "Upgrade Plan," then select Pro Max. This is the plan that gives you access to Claude Code, which is the tool you'll use to build and configure everything. The regular Pro plan ($20/month) does not include Claude Code.

Get your API key

Go to console.anthropic.com. Click "API Keys" in the left sidebar. Click "Create Key." Name it something like "VESPR-EA". Copy the key and save it somewhere safe. You will need this key later when connecting n8n to Claude. Never share this key publicly.

Important

Your API key is like a password. Anyone who has it can use your Claude account and run up charges. Store it in a password manager or secure note. Never paste it into public code, social media, or shared documents.

Section 44 Installing Claude Code

Claude Code is a command-line tool that lets you talk to Claude directly from your computer's terminal. Instead of typing in a chat window, you give Claude instructions and it writes code, creates files, and builds automations for you. Think of it as having a developer sitting next to you who does exactly what you ask.

What is a Terminal?

The terminal (also called command line) is a text-based interface on your computer. Instead of clicking buttons, you type commands. On Mac, it's the app called "Terminal." On Windows, it's "Command Prompt" or "PowerShell."

Installation Steps

Open your Terminal

On Mac: Press Cmd + Space, type "Terminal", press Enter.

On Windows: Press Win + R, type "cmd", press Enter.

Install Node.js (if you don't have it)

Claude Code needs Node.js to run. Check if you have it by typing:

node --version

If you see a version number (like v20.x.x), you're good. If you see an error, go to nodejs.org, download the LTS version, and install it. Then restart your terminal.

Install Claude Code

Type this command in your terminal and press Enter:

npm install -g @anthropic-ai/claude-code

Wait for it to finish. You'll see some text scrolling. When you get back to the prompt, it's done.

Launch Claude Code

Navigate to a folder where you want to build your project, then start Claude Code:

mkdir ~/vespr-ea
cd ~/vespr-ea
claude

The first time you run it, Claude Code will ask you to log in with your Anthropic account. Follow the prompts to authenticate. Once logged in, you'll see the Claude Code interface where you can type natural language instructions.

Your First Claude Code Command

Try this to confirm everything is working:

Type this into Claude Code

Create a file called test.txt that says "VESPR EA is online" and confirm it was created.

Claude will create the file and confirm. If that works, you're ready to build. Delete the test file and move on.

Section 55 Setting Up n8n

n8n is the automation engine that connects all your channels to Claude. Think of it as the plumbing. When a new email arrives, n8n catches it, sends it to Claude for analysis, and pushes the result to your dashboard. No code required - it's all visual drag-and-drop.

Create your n8n Cloud account

Go to n8n.io and click "Get Started Free." Sign up with your email. Choose the Starter plan ($24/month). This gives you enough workflow executions for a personal EA system.

Log into your n8n dashboard

Once your account is created, you'll land on the n8n workflow editor. This is where you'll build all your automations. The interface shows a canvas where you drag nodes (each node is an action, like "read email" or "send to Claude").

Add your Claude API credentials to n8n

Click the gear icon (bottom left), then "Credentials." Click "Add Credential" and search for "Anthropic." Paste the API key you saved from Section 3. Name it "Claude-VESPR." Click Save.

Why n8n and not Zapier or Make?

n8n gives you full control, self-hosting options, and no per-task pricing that blows up when your volume grows. At scale, Zapier would cost 10-20x more. n8n also handles complex branching logic and AI nodes natively.

Section 66 Connecting Your Email

Email is usually the highest-volume channel. This section connects Gmail to n8n so every incoming email gets captured and sent to Claude for analysis.

Setting Up Gmail API Access

Go to Google Cloud Console

Open console.cloud.google.com. Sign in with the Gmail account you want to monitor. Click "Select a project" at the top, then "New Project." Name it "VESPR-EA" and click Create.

Enable the Gmail API

In your project, go to "APIs & Services" then "Library." Search for "Gmail API" and click Enable. Then search for "Google Calendar API" and enable that too (you'll need it later).

Create OAuth credentials

Go to "APIs & Services" then "Credentials." Click "Create Credentials" then "OAuth client ID." Select "Web application." Under Authorized redirect URIs, add your n8n OAuth callback URL (found in n8n under Settings > Credentials when you add a Google credential). Click Create. Copy the Client ID and Client Secret.

Connect Gmail in n8n

In n8n, create a new workflow. Add a "Gmail Trigger" node. Click to configure it. Add a new Google credential using the Client ID and Secret from above. Authorize the connection (Google will ask you to log in and grant access). Set the trigger to fire on "New Email Received." Save.

Building the Email Processing Workflow

Now create the workflow that processes each email through Claude. In n8n, your workflow should look like this:

Gmail Trigger → Anthropic (Claude) Node → Webhook (Dashboard)

For the Anthropic node, use this prompt configuration:

Claude Node System Prompt

You are an AI executive assistant for [YOUR NAME]. You analyze incoming emails and produce a JSON response with the following fields: urgency_score (0-100), priority ("urgent", "action", or "fyi"), summary (1-2 sentences), ai_notes (array of 3-5 strategic observations), draft_reply (a response written in [YOUR NAME]'s voice - direct, decisive, no fluff). Consider the sender's relationship to the business, any deadlines mentioned, financial implications, and whether a response is needed today.

Pro tip

Add examples of your actual emails and how you'd respond to the system prompt. The more examples Claude has of your voice and decision-making style, the better the drafts will be. We cover this in detail in Section 12.

Section 77 Connecting Slack

Slack messages from your team get pulled into the same unified inbox. Here's how to connect it.

Create a Slack App

Go to api.slack.com/apps. Click "Create New App" then "From scratch." Name it "VESPR EA" and select your workspace. Click Create App.

Set permissions (OAuth Scopes)

In the sidebar, click "OAuth & Permissions." Scroll to "Bot Token Scopes" and add: channels:history, channels:read, chat:write, im:history, im:read, users:read. These let your app read messages and send replies.

Install the app to your workspace

Click "Install to Workspace" at the top of the OAuth page. Authorize it. Copy the "Bot User OAuth Token" that appears. You'll need this for n8n.

Enable Event Subscriptions

In the sidebar, click "Event Subscriptions." Turn it on. For the Request URL, you'll use an n8n webhook URL. In n8n, add a "Webhook" node, copy its URL, and paste it here. Under "Subscribe to bot events," add message.channels and message.im. Save.

Build the Slack workflow in n8n

Create a new workflow: Webhook (Slack events) → Filter (ignore bot messages) → Anthropic (Claude) Node → Webhook (Dashboard). Use the same Claude system prompt as email, but add "source: slack" to the context.

Section 88 Connecting WhatsApp

WhatsApp is where many business conversations happen, especially with clients and partners. Here's how to pipe those messages into your EA.

Set up WhatsApp Business API via Meta

Go to developers.facebook.com. Create a Meta Developer account if you don't have one. Create a new App, select "Business" type. Add the "WhatsApp" product. You'll get a test phone number and API credentials to start.

Configure the webhook in n8n

In n8n, add a Webhook node. Copy its URL. Back in the Meta Developer portal, go to WhatsApp > Configuration > Webhook. Paste the n8n webhook URL. Set the Verify Token to something unique (like "vespr-ea-2026"). Subscribe to "messages" events.

Build the WhatsApp workflow

Same pattern: Webhook (WhatsApp) → Parse message body → Anthropic (Claude) Node → Webhook (Dashboard). To send replies back, add a "WhatsApp Business Cloud" node at the end that fires when you approve a draft.

Alternative: WhatsApp via n8n Community Node

If the official API feels complex, n8n has community nodes like "WhatsApp Web" that connect through a QR code scan, just like WhatsApp Web on your computer. This is simpler to set up but less reliable for production. Start here if you want to test quickly, then upgrade to the official API later.

Section 99 Connecting iMessage

iMessage is the trickiest channel because Apple doesn't offer an official API. Here are two approaches depending on your comfort level.

Option A: BlueBubbles (Recommended)

BlueBubbles is an open-source app that runs on a Mac and exposes your iMessages through an API. This is the most reliable approach.

Install BlueBubbles on a Mac

Go to bluebubbles.app and download the server app. This needs to run on a Mac that stays on (can be an old Mac Mini or your main machine). Install it and sign in with your Apple ID.

Enable the API and webhooks

In BlueBubbles settings, enable the REST API. Set up a webhook URL pointing to your n8n webhook node. BlueBubbles will send a payload every time you receive a new iMessage.

Build the iMessage workflow in n8n

Same pattern as the others: Webhook (BlueBubbles) → Anthropic (Claude) Node → Webhook (Dashboard). To send replies, use the BlueBubbles API to send messages back through iMessage.

Option B: Manual Forwarding

If you don't want to run BlueBubbles, you can set up iMessage forwarding to your email. On your Mac, open Messages > Settings > iMessage > enable "Text Message Forwarding." Then forward those to your Gmail, where they'll be caught by your email workflow. This is less elegant but requires zero setup.

Section 1010 Connecting Google Calendar

Your AI EA needs to see your calendar to schedule meetings, identify conflicts, and plan your day. This connection also powers the daily briefing's calendar section.

Use the same Google Cloud project from Section 6

You already enabled the Google Calendar API. Now go to n8n and add a "Google Calendar" node. Use the same Google OAuth credentials you created for Gmail. Authorize calendar access.

Create the daily calendar fetch workflow

Create a new workflow with a Schedule Trigger (set to 6:00 AM daily) → Google Calendar node (Get Events, time range: today) → Anthropic (Claude) Node → Webhook (Dashboard).

Configure the scheduling workflow

For the "Schedule Meeting" feature, create a separate workflow: Webhook (from dashboard) → Google Calendar (Check availability) → Claude (Find 3 best times) → Gmail (Send invite email) → Google Calendar (Create hold).

Claude Prompt for Calendar Analysis

Here is my calendar for today: {{calendar_events}}. Analyze my day and produce: (1) a natural language briefing of my day as if you're my EA giving me a 90-second morning update, (2) identify any conflicts or tight transitions, (3) note which blocks are protected (deep work, personal) and should not be scheduled over, (4) suggest the 3 best open slots for a new 30-minute meeting.

Section 1111 Building the AI Brain

This is the core of the system. The AI Brain is a single Claude prompt that takes any incoming message and outputs a structured analysis with urgency score, notes, and a draft reply. Here's the complete prompt you'll use.

The Master Classification Prompt

This goes in the Anthropic node in every n8n workflow (email, Slack, WhatsApp, iMessage). It's the same prompt for all channels.

Master System Prompt
You are the AI Executive Assistant for [YOUR NAME].
You manage communications across all channels for
[BRIEF DESCRIPTION OF YOUR BUSINESSES].

Your job is to analyze each incoming message and return
a JSON object with these fields:

{
  "from": "sender name",
  "source": "email | slack | whatsapp | sms",
  "urgency_score": 0-100,
  "priority": "urgent | action | fyi",
  "subject": "short subject line",
  "summary": "1-2 sentence summary",
  "ai_notes": [
    "strategic observation 1",
    "strategic observation 2",
    "strategic observation 3",
    "recommendation"
  ],
  "urgency_reason": "why this urgency score",
  "draft_reply": "response in [YOUR NAME]'s voice"
}

URGENCY SCORING GUIDE:
- 90-100: Money on the table, deadlines today,
  revenue-impacting situations
- 70-89: Important business decisions, key
  relationships, time-sensitive opportunities
- 50-69: Needs a response today but not immediately
- 20-49: Can wait 24-48 hours
- 0-19: Informational only, no response needed

VOICE GUIDELINES FOR DRAFT REPLIES:
- Direct and decisive. No filler words.
- Lead with the action or decision.
- Keep it under 4 sentences when possible.
- Use first name only for the greeting.
- Never use em dashes. Use hyphens only.
- Sound like a founder, not a customer service rep.

BUSINESS CONTEXT:
[INSERT YOUR BUSINESS DETAILS, KEY CLIENTS,
ONGOING DEALS, TEAM MEMBERS, AND ANY CONTEXT
THAT HELPS CLAUDE MAKE BETTER DECISIONS]
The more context, the better

The "Business Context" section is what makes this system actually useful. Include your current clients, active deals, team members' names and roles, ongoing projects, and any standing priorities. Update this monthly as things change.

Section 1212 Training Your AI On Your Voice

The difference between a generic AI assistant and one that sounds exactly like you is training data. Here's how to give Claude your voice, your decision-making patterns, and your business context so every draft reply sounds like it came from you.

Step 1: Collect 20 Real Conversations

Go through your email, Slack, and WhatsApp. Find 20 messages where you responded in a way that represents your style. Copy both the incoming message and your actual reply. These should cover different scenarios:

  • 5 responses to clients/customers
  • 5 responses to team members
  • 3 responses to new business inquiries/leads
  • 3 responses to partners or investors
  • 2 responses to vendors or service providers
  • 2 responses declining or saying no to something

Step 2: Build Your Voice Profile

Open Claude Code in your project folder and use this prompt:

Claude Code Prompt

I'm going to paste 20 email/message exchanges. I want you to analyze my writing style and create a "voice profile" that captures: my tone, my sentence structure, how I open messages, how I close them, phrases I use often, my decision-making speed, what I'm direct about vs. where I'm more careful. Save this as a file called voice-profile.md. Here are my messages: [PASTE YOUR 20 CONVERSATIONS]

Claude will analyze your patterns and create a detailed profile. Read through it and edit anything that doesn't feel right. This file becomes part of the system prompt.

Step 3: Add the Voice Profile to Your System Prompt

Take the voice-profile.md content and add it to the "Voice Guidelines" section of your master prompt from Section 11. Now every draft reply will match your actual communication style.

Step 4: Refine Over Time

Every time you edit a draft before sending, you're training yourself on what Claude gets wrong. After a week, collect the edits you made and update the voice profile:

Claude Code Prompt

Here are 10 draft replies you wrote for me and the edits I made before sending. Analyze what you got wrong and update my voice-profile.md accordingly. Focus on patterns - don't just memorize individual corrections. [PASTE ORIGINAL DRAFTS AND YOUR EDITED VERSIONS]

How good does it get?

After 2-3 refinement cycles (about 2-3 weeks of daily use), most people find they're approving 80-90% of drafts without edits. The AI gets better every week because you're continuously training it on the gap between what it writes and what you actually send.

Section 1313 Deploying the Dashboard

The dashboard is the frontend where you review everything each morning. The HTML file included with this guide is your starting point. Here's how to make it live and connect it to your n8n workflows.

Step 1: Download the Dashboard Code

Get the Dashboard Source Code

The complete VESPR OS dashboard is open source. Download the code, customize it for your business, and deploy it.

Download from GitHub
Click "Code" then "Download ZIP" on GitHub. No git knowledge required.

Open the dashboard file

Unzip the download and open index.html in your browser. That's the full interactive dashboard with demo data. You'll connect it to your real data next.

Customize it with Claude Code

Open Claude Code in the dashboard folder and tell it to customize the dashboard for your business:

Claude Code Prompt

Read the index.html dashboard file. I want to customize it for my business. My name is [YOUR NAME]. I run [YOUR BUSINESSES]. My team members are [NAMES AND ROLES]. Replace all the demo data with placeholder data that matches my actual business. Keep all the functionality the same - just update the content.

Step 2: Connect to Real Data via n8n

To make the dashboard pull live data instead of demo data, you'll add a small backend. Use Claude Code to build it:

Claude Code Prompt

I need to connect this dashboard to my n8n workflows. Create a simple backend using Node.js and Express that: (1) receives webhook data from n8n containing processed messages, (2) stores them in a local JSON file, (3) serves them to the dashboard via an API endpoint, (4) handles the "approve and send" action by calling back to n8n to send the reply. Include a fetch call in the dashboard that loads messages from the API instead of the hardcoded array.

Step 3: Deploy to Vercel (Free)

Push your code to GitHub

In Claude Code, tell it to initialize a git repo and push to GitHub:

Initialize a git repo here, create a .gitignore
for node_modules, and give me the commands to push
this to a new GitHub repo called vespr-ea-dashboard.

Connect Vercel

Go to vercel.com, sign in with GitHub. Click "New Project," import your vespr-ea-dashboard repo. Vercel will auto-detect the setup and deploy it. You'll get a live URL like vespr-ea-dashboard.vercel.app.

Update n8n webhook URLs

Go back to your n8n workflows and update all the "Webhook (Dashboard)" nodes to point to your Vercel URL. Now your live dashboard receives real processed messages.

Section 1414 Daily Workflow: How to Use It

Here's your new morning routine. This should take 5-10 minutes total.

Every Morning (6:47 AM or whenever you start)

Open your dashboard

Go to your Vercel URL or bookmark it. The Daily Briefing screen loads first with your "Day at a Glance" summary.

Read the AI briefing

This is your 90-second morning download. It tells you what's urgent, what's important, and the priority order for your morning. It's personalized to your businesses and current deals.

Review and approve urgent drafts

Scroll down to the Urgent section. Read the AI's draft reply for each one. If it's good, click "Approve & Send." If it needs tweaking, click "Edit Draft," make your changes, then approve. Most mornings you'll approve 80%+ without edits.

Handle action items

Review the Action Required section. These need a response today but aren't emergencies. Approve drafts or open the full conversation in the Inbox to review context before responding.

Check your calendar

Click Calendar in the sidebar to see today's schedule. Click any event for details. Use "Schedule Meeting" if you need to book something new. The AI finds open slots and sends the invite for you.

Review tasks

Click Tasks to see what your AI has delegated, what's due today, and what's coming this week. Check off completed items. Click "See conversation" on any task to jump to the related message.

The 5-minute rule

If your daily review takes more than 10 minutes, something needs tuning. Either your urgency thresholds are too sensitive (too many things marked urgent), your draft quality needs improvement (see Section 12 for training), or you're reading FYI messages instead of skipping them. FYI means no response needed - trust the AI's classification.

Section 1515 Troubleshooting + FAQ

Common Setup Questions

Q: Do I need to know how to code?

No. Claude Code writes the code for you. You describe what you want in plain English, and it builds it. If something breaks, you describe the problem and Claude fixes it. That said, being comfortable with copy-pasting commands into a terminal is the minimum required skill.

Q: Can I use this with Outlook instead of Gmail?

Yes. n8n has a Microsoft Outlook / Office 365 node. The setup process is similar - you create an app in Azure Portal instead of Google Cloud Console, get OAuth credentials, and connect. Use the same Claude prompts for processing.

Q: What if I don't use Slack?

Skip Section 7. The system works with whatever channels you actually use. You might only need Email + WhatsApp. Or Email + iMessage. Each channel is independent - add or remove them without affecting the others.

Q: How long until the AI drafts sound like me?

Day 1: about 60% accuracy. After your first voice profile training (end of week 1): about 75%. After your first refinement cycle (end of week 2): about 85-90%. Most people stop editing drafts regularly within 3 weeks.

Q: What happens if n8n goes down?

Your messages are still in Gmail, Slack, etc. You just don't get the AI processing until n8n is back. n8n Cloud has 99.9% uptime. When it recovers, it processes any queued messages automatically.

Q: Can I add more channels later (like Twitter DMs, LinkedIn, etc.)?

Yes. n8n has nodes for almost every platform. The pattern is always the same: Trigger node (catches the message) → Claude node (processes it) → Dashboard webhook (displays it). Once you've built one channel, adding another takes 15 minutes.

Q: How do I handle sensitive/confidential messages?

All processing happens through your own API key on Anthropic's servers with their privacy policy (they don't train on API data). For extra security, you can self-host n8n instead of using Cloud, which keeps everything on your own servers.
